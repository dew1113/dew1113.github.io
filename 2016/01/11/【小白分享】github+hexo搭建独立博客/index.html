前言

什么是小白

原谅我一直以为小白指的是只会开关电脑的人， 但是经历了搭建这个博客的心酸血汗史后，我才发现，我连大神眼中的小白都谈不上啊！那些经验分享赤裸裸地碾压了我140的智商！ （/TДT)/ 我十分气愤，于是要开启真正的小白经验分享，只要你会开关电脑，就一定能看得懂我写的文章。

其实我本来也是想写个上千字的搭建部署以及修改的详细步骤的， 但是发现网上资料都已经很详细了，只是有些人安装写的详细，有些人主题写的详细。所以呢，我就总结一些我遇到的问题吧。

(◕ˇ∀ˇ◕)我要进入正题啦

一、部署时执行：hexo deploy时页面没有反应

解决方法：

  1. 检查一下站点配置_config.yml文件，最下面deploy标签下的几个属性冒号后必须空一格空一格空一格！重要的事情说三遍，因为你不空格也不报错，真是反人类啊~！我的设置如下，可以参考一下。
    deploy: type: git repository: git@github.com:dew1113/dew1113.github.io.git branch: master

  2. 在当前目录下安装一下部署文件
    $ npm install hexo-deployer-git --save

二、在本地可以运行查看到，但是一打开页面就是404

解决方法：

  1. 如果你是使用的自己的一级域名的话，检查仓库里是否增加了CNAME文件，这个文件最好不要在仓库内手动加进去，我每次手动加入，再更新时总是莫名被删除。到我的github仓库，点击右下角的「Download ZIP」，下载源文件，解压，找到CNAME文件，用记事本打开，将chenlu.win修改成你的域名，放进Hexo\source目录下，用hexo命令提交上去。
    $ hexo d -g

三、本地运行无法访问

解决方法：

  1. 执行hexo sever之后会显示running at http://0.0.0.0:4000/，但是在浏览器直接尝试http://0.0.0.0:4000/ 是没办法访问的。要输入http://localhost:4000/ 才可以。

四、NexT主题下如何添加多说评论功能

具体步骤：

  1. 登录多说http://duoshuo.com/首页，选择“我要安装”。
  2. 创建站点，填写表单。多说域名这一栏填写的即是你的duoshuo_shortname
  3. 创建站点完成后再站点配置文件（根目录下的_config.yml）中新增一个duoshuo_shortname字段，值设置成上一步中的值。
  4. OK啦~ 添加评论就是SO EASY~(￣▽￣)/

五、如何更改网页标头上的小图标favicon

这个就更easy啦，制作一个icon图标，啥？你说你不会做？那就选择一张你喜欢的图片，用画图打开，该裁剪裁剪，然后另存为favicon.icon到根目录的source下面。然后
$ hexo g --d

不过这个可能不会立马更新出来，一开始我以为设置失败了呢，放了几分钟后就自己出来了。

六、编写markdown文章使用什么编辑器比较好呢

一开始我用了个markhua的在线编辑，可以边写边预览，但是总抽风，写二十来个字就写不出来了，也不知道为啥，后来就下载了一个MarkdownPad2，中文课破解的，我还没破解。。。因为找不到某个文件，但是可以把资源和方法贴出来，大家自己安装破解一下，我个人觉得挺好用的。

MarkdownPad 2下载：http://pan.baidu.com/s/1i3ULvfB
