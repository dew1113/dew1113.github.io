【学习记录】DAY24-AJAX

发表于 2016-03-14   |   分类于 技术   |  

1.ajax 是什么？有什么作用？

AJAX = Asynchronous JavaScript and XML（异步的 JavaScript 和 XML 通过在后台与服务器进行少量数据交换，AJAX 可以使网页实现异步更新。这意味着可以在不重新加载整个网页的情况下，对网页的某部分进行更新。

2.前后端开发联调需要注意哪些事情？后端接口完成前如何 mock 数据？(npm install -g server-mock) 知识点视频-如何 mock 数据

注意事项：

  • 在开发之前规定好接口文档，指明由谁来撰写和维护；
  • 如果接口的信息改动需要互相确认改动信息；
  • 确定接口数据传输类型，是json/xml还是jsonp；
  • 定义数据管理和归属权，是属于前端管理还是后端管理；
  • 确定数据传输方式，前后端之间是否有中间层。

后端接口完成前如何mock数据： 使用模拟数据生成器：Mock.js

3.点击按钮，使用 ajax 获取数据，如何在数据到来之前防止重复点击?

  1. 点击完以后，将按钮隐藏。display:none,但是这个做法用户体验很不好，本来有个按钮在那儿，一点就没了，用户要是想再点都没东西可点了。。
  2. 如果点击按钮是input类型的，可以用户点击完以后增加一个disabled属性：<input type='button' id="btn-register" value='注册' disabled='disabled'>，不过这个方法同上，可以防止重复点击，但是用户体验不好，不推荐。
  3. 【推荐方法】使用一个开关。自定义一个变量 var turn=false,事件绑定的时候，打开开关turn=true,调用函数，函数调用完了，再关闭开关turn =false，用户体验良好，还可以有效地防止重复点击。
