问答

\d，\w,\s,[a-zA-Z0-9],\b,.,*,+,?,x{3},^,$分别是什么?

  • \d:查找数字
  • \w: 查找单词字符（数字，字母，下划线）
  • \s:查找空白字符（空格、tab、换行、回车）
  • [a-zA-Z0-9]:查找0-9十个数字，a-z26个小写字母和A-Z的26个大写字母
  • \b:匹配单词边界
  • .: 查找单个字符，除了换行和回车符
  • *: 匹配出现0次或者多次（任意多次）
  • +: 匹配出现一次或多次（至少一次）
  • ?: 匹配出现0次或者1次（至多1次）
  • X{3}: 匹配x出现3次
  • ^ : 匹配以什么开头
  • $: 匹配以什么结尾

【学习记录】DAY20-闭包-定时器

发表于 2016-03-03   |   分类于 技术   |  

问题

什么是闭包? 有什么作用

  • 闭包就是能够读取其他函数内部变量的函数。由于在Javascript语言中，只有函数内部的子函数才能读取局部变量，因此可以把闭包简单理解成”定义在一个函数内部的函数”。所以，在本质上，闭包就是将函数内部和函数外部连接起来的一座桥梁。
  • 作用：一是读取函数内部的变量，二是让这些变量的值始终保持在内存中。

setTimeout 0有什么作用

因为JavaScript 是单线程执行的，也就是无法同时执行多段代码，当某一段代码正在执行的时候，所有后续的任务都必须等待，形成一个队列，一旦当前任务执行完毕，再从队列中取出下一个任务。这也常被称为 “阻塞式执行”。所以一次鼠标点击，或是计时器到达时间点，或是 Ajax 请求完成触发了回调函数，这些事件处理程序或回调函数都不会立即运行，而是立即排队，一旦线程有空闲就执行。假如当前JavaScript 进程正在执行一段很耗时的代码，此时发生了一次鼠标点击，那么事件处理程序就被阻塞，用户也无法立即看到反馈，事件处理程序会被放入任务队列，直到前面的代码结束以后才会开始执行。如果代码中设定了一个 setTimeout，那么浏览器便会在合适的时间，将代码插入任务队列，如果这个时间设为 0，就代表立即插入队列，但不是立即执行，仍然要等待前面代码执行完毕。所以 setTimeout 并不能保证执行的时间，是否及时执行取决于 JavaScript 线程是拥挤还是空闲。

作用

  • 调整事件的发生顺序。setTimeout(f,0)一大应用就是可以调整事件的发生顺序。比如，网页开发中，某个事件先发生在子元素，然后冒泡到父元素，即子元素的事件回调函数，会早于父元素的事件回调函数触发。如果，我们先让父元素的事件回调函数先发生，就要用到setTimeout(f, 0)。

  • 分割耗时任务。setTimeout一个很关键的用法就是分片，如果一段程序过大，我们可以拆分成若干细小的块。由于setTimeout(f,0)实际上意味着，将任务放到浏览器最早可得的空闲时段执行，所以那些计算量大、耗时长的任务，常常会被放到几个小部分，分别放到setTimeout(f,0)里面执行(分片塞入队列)，这样即使在复杂程序没有处理完时，我们操作页面，也是能得到即时响应的。其实就是将交互插入到了复杂程序中执行。

问题

基础类型有哪些？复杂类型有哪些？有什么特征？

基础类型：字符串、数字、布尔值、null、undefined。指的是保存在栈内存中的简单数据段

复杂类型：数组，对象，函数，正则表达式。复杂值可以由很多不同类型的javascript对象组成。复杂对象其在内存中的大小是未知的，因为复杂对象可以包含任何值，而不是一个特定的已知值。

存储方式:

栈存储

　　因为基础类型占据空间固定，是简单的数据段，为了便于提升变量查询速度，将其存储在栈(stack)中

堆存储

　　由于复杂类型的大小会改变，所以不能将其存放在栈中，否则会降低变量查询速度，因此其存储在堆(heap)中，存储在变量处的值是一个指针，指向存储对象的内存处。

【数组方法里push、pop、shift、unshift、join、split分别是什么作用。（*）】

push():在数组最后一位添加元素,返回添加元素后的长度

pop():删除数组最后一个元素，返回该元素

shift():移除数组左边第一个元素，返回该元素

unshift():在数组最前面添加元素，返回添加元素后的长度

join():以参数为连接符，连接数组形成字符串

split():以参数为分隔符，将字符串拆分为字符串数组

splice():array.splice(start, deleteCount[, item1[, item2[, ...]]])用新元素替代旧元素，以此修改数组的内容。返回由被删除的元素组成的一个数组。如果只删除了一个元素，则返回只包含一个元素的数组。如果没有删除元素，则返回空数组。

【1.负边距在让元素产生偏移时和position:relative有什么区别？】

负边距使元素产生偏移时，不会脱离文档流，元素位置依然存在，所以会对后面的元素位置产生影响。
而positon:relative会使得元素脱离文档流，因为对其他元素的位置并不会产生影响。

【2.使用负margin形成三栏布局有什么条件？】

  • 父容器要设置左右padding为左右边栏预留出空位；
  • 使用position:relative将浮动的左右边栏定位到两侧。
